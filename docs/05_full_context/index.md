# Full Context

This section contains deeper explanations, SOPs, rationale, and architecture notes.

Use this when redesigning or auditing the system.

Do not use Full Context as the normal daily operating surface.

## Current useful guide

- `sops/user-week-rollaround-guide.md`

## Main principle

Weekly Board = human surface.  
Route Cards = operational contracts.  
Digital Support Index = links.  
State Sources = truth.  
Evidence Sources = proof.  
Full Context = redesign/audit.
