# CLIENT-SITES-GOV — Static Sites Offer Governance

## When

Use when the static-sites/business front is active.

## Purpose

Convert web-building capability into a sellable, repeatable offer.

## Inputs

- Offer draft
- Demo site
- SOP
- Prospect list
- Pricing or scope notes
- Client follow-up route

## Steps minimum

1. Choose one commercial surface: offer, demo, prospect, delivery SOP.
2. Produce or improve one external-facing artifact.
3. Send/share/contact if possible.
4. Update next pointer.

## Output

Offer/demo/prospect/delivery system advanced.

## Tick valid when

The business front becomes more sellable, visible, or repeatable.

## Stop rule

Do not build an agency operating system before validating the offer.
