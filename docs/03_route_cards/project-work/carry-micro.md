# CARRY-M — Carry Micro Update

## When

Use Monday, Wednesday, Friday, or after a meaningful project movement.

## Purpose

Keep active fronts legible without doing a full project review.

## Inputs

- Carry State
- Recently touched artifact
- Weekly board annotation
- Current blocker, if any

## Steps minimum

1. Identify what changed.
2. Update status only if needed.
3. Add or refresh next pointer.
4. Mark blocker or parked reason if relevant.
5. Stop.

## Output

A small Carry State update.

## Tick valid when

The changed front can be restarted from its current state.

## Stop rule

Three minutes by default. If it needs more, escalate to CARRY-W or project triage.
