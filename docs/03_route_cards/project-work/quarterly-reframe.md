# QUARTERLY-REFRAME — Seasonal / Quarterly Reframe

## When

Use during a scheduled meta anchor, not during ordinary daily execution.

## Purpose

Review fronts and choose which assets should be converted into visible value this season.

## Inputs

- Current fronts
- Carry State
- Recent outputs
- Stalled projects
- External opportunities
- Personal constraints

## Steps minimum

1. List major fronts.
2. Identify which fronts are already mature enough to convert.
3. Choose 2-3 seasonal bets.
4. Decide what remains maintenance/incubation/not-now.
5. Define first 14-day breakthrough.
6. Write boundaries.

## Output

Seasonal orientation and first 14-day frame.

## Tick valid when

The next two weeks can be planned from the seasonal decision.

## Stop rule

Do not turn the reframe into a plan for every project.
