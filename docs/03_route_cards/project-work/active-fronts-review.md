# ACTIVE-FRONTS — Active Fronts Review

## When

Use inside WEEK-GOV, CARRY-W, or a 14-day review.

## Purpose

Make sure the current active fronts match the week’s real capacity.

## Inputs

- Carry State
- Weekly board
- 14-day breakthrough, if active
- External deadlines
- Personal constraints

## Steps minimum

1. Name active fronts.
2. Mark which front is primary.
3. Mark which fronts are support-only.
4. Mark which fronts are not-now.
5. Confirm the next output for the primary front.

## Output

A clean active-front set.

## Tick valid when

The week has a primary front and no more active fronts than can realistically be touched.

## Stop rule

Do not create new fronts during this review unless one replaces an existing active front.
