# SLEEP-LOG — Sleep Signal

## When

Use in the morning or during BODY-GOV.

## Purpose

Keep sleep visible enough to interpret energy, recovery, and planning quality.

## Inputs

- Sleep time
- Wake time
- Subjective sleep quality
- Alcohol/caffeine if relevant
- Gym/recovery if relevant

## Steps minimum

1. Record sleep and wake time if known.
2. Mark estimate if exact time is unknown.
3. Add one quality signal if useful.
4. Route any important constraint to BODY-GOV or GYM-GOV.

## Output

Sleep signal recorded.

## Tick valid when

Sleep is recorded or explicitly marked unknown.

## Stop rule

Do not overanalyze sleep during routine logging.
