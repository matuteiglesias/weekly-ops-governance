# Linking Rules

## Rule 1 — Stable IDs first

Use stable route IDs in links and headings.

Examples:

- `FIN-W`
- `CARRY-W`
- `GYM-GOV`
- `HOUSE-GOV`
- `CAPTURE-GOV`
- `WEEK-GOV`
- `EXTERNAL`

## Rule 2 — The index links out, it does not duplicate

The Digital Support Index should not copy full route card content, checklist content, or full manual context.

It should provide enough description to choose the right link.

## Rule 3 — Every route should link to four things

Each route should try to link to:

1. route card;
2. governance contract;
3. state source;
4. evidence target.

Checklist/manual links are added when they exist.

## Rule 4 — Placeholder links are explicit

If a real sheet, calendar, or app link is not known yet, use:

- `TODO: add real link`
- `LOCAL: path pending`
- `EXTERNAL: URL pending`

Do not invent fake URLs.

## Rule 5 — Broken links are operational debt

A broken link is not just documentation drift. It increases execution friction.

Run link health checks during maintenance or support index review.
