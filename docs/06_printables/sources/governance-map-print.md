# Governance Map — Printable Reference

> Use this as a light architecture reference. It is not the weekly board.

---

## Routing Architecture

```text
Weekly Board
  -> route ID / checkbox / annotation
  -> governance contract
  -> route card
  -> digital support index
  -> state source
  -> evidence / next pointer
````

---

## Core Routes

| ID       | Domain            | Use                                          |
| -------- | ----------------- | -------------------------------------------- |
| WEEK-GOV | Weekly governance | Frame the week and active routes             |
| BOOT     | Daily execution   | Start the day without reconstructing context |
| MAINT    | Maintenance       | Reduce friction and prepare execution        |
| FOCUS    | Deep work         | Produce evidence                             |
| CLOSE    | Daily close       | Prevent context loss                         |
| NEXT     | Continuity        | Make restart cheap                           |

## Rule

The weekly board shows IDs.
The route cards define action.
The support index gives links.
State sources hold truth.
Evidence proves movement.
