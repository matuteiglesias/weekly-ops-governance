# MAINT-GOV — Maintenance Contract

## Purpose

Use maintenance hours to reduce friction, prepare execution, and prevent small disorder from spreading into the week.

## Contract

Maintenance is valid when it:

- prepares the next block;
- closes a small loop;
- cleans a broken link/path/context;
- reduces house/body/admin friction;
- updates state cheaply;
- prevents future rework.

## Inputs

- Maintenance quicklist
- Weekly board
- Helper pages
- Broken links
- Small operational debts
- Inboxes or loose notes
- Route cards if applicable

## Outputs

- Cleaner context
- Prepared block
- Closed loop
- Repaired link
- Updated small state
- Deferred item with next pointer

## Tick valid when

One real source of friction was reduced or explicitly parked with a next pointer.

## Stop rule

Maintenance is not open-ended debugging, refactoring, project redesign, or a productivity sink.
