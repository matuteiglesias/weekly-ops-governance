# EXTERNAL-GOV — Externalization Contract

## Purpose

Ensure that work leaves the private system and reaches people, platforms, clients, recruiters, readers, or reviewers.

## Contract

Externalization is valid when an external surface changes.

Examples:

- message sent;
- post published;
- recruiter answered;
- client contacted;
- feedback requested;
- demo shared;
- report delivered.

## Inputs

- Draft
- Contact list
- Publication target
- Recruiter/client/media thread
- Artifact ready enough to send

## Outputs

- Message sent
- Post published
- Feedback requested
- Client/recruiter contacted
- Demo shared
- Artifact delivered

## Tick valid when

At least one external surface changed.

## Stop rule

Do not keep polishing if the artifact is already good enough for the intended external action.
