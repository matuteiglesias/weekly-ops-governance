# FOCUS-GOV — Focus Work Contract

## Purpose

Convert focus time into observable progress.

## Contract

A focus block is valid when it has:

- one selected front;
- one expected output;
- one time boundary;
- one stop condition;
- evidence or a next pointer.

## Inputs

- Active front
- Selected route, if any
- Relevant artifact, repo, doc, sheet, or message
- Carry State / next pointer
- Current energy and constraints

## Outputs

- Draft
- Commit
- Report
- Message
- Decision
- Deployed artifact
- Updated state
- Next pointer

## Tick valid when

The block leaves an observable artifact, state update, external action, or restartable next pointer.

## Stop rule

Do not use a focus block to redesign the whole system unless the block was explicitly assigned to design.
